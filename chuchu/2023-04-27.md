## 2023-04-27 (목)

### JPA
    - 낙관적 락 : @Version을 적용한 필드를 이용해서 버전을 관리한다. 엔티티가 조회될 때 해당 필드의 값이 자동으로 업데이트됨
    - 비관적 락 : 항상 트랜잭션 충돌이 난다고 가정. 가장 자주 사용하는 방법은 조회하는 로우에 락을 거는 select for update.
                락이 걸린 로우는 다른 트랜잭션에서 사용할 수 없음
    - 2차 캐시 : 어플리케이션 전반에 걸쳐서 사용할 수 있는 캐시로, 어플리케이션이 시작할 떄 생성되고, 종료될 때 함께 사용이 종료된다.
                2차 캐시를 사용하면 엔티티 조회시 1차 -> 2차 -> 데이터베이스 순서대로 조회하고, 2차 캐시에서 엔티티를 조회하면 엔티티를 복사해서 반환한다.
